# Memory Management — Implementation Instructions

Apply these rules across the entire khayal codebase. These are not optional optimisations — they are correctness requirements.

---

## 1. Embedding search — stream and compare, never load all into memory

```go
// NEVER
rows, _ := db.Query(`SELECT embedding FROM chunks`)
var all [][]float32
for rows.Next() {
    var emb []float32
    rows.Scan(&emb)
    all = append(all, emb)
}

// ALWAYS — fixed-size heap, stream one row at a time
rows, _ := db.Query(`SELECT note_path, embedding FROM chunks`)
defer rows.Close()

type scored struct {
    path  string
    score float32
}
heap := make([]scored, 0, topK)

for rows.Next() {
    var path string
    var embBytes []byte
    rows.Scan(&path, &embBytes)
    emb := bytesToFloat32(embBytes)
    score := cosineSimilarity(queryEmb, emb)
    updateMinHeap(&heap, scored{path, score}, topK)
}
```

Peak memory = `topK * 3KB` regardless of vault size.

## 2. Ollama responses — stream decode, never ReadAll

```go
// NEVER
data, _ := io.ReadAll(resp.Body)
json.Unmarshal(data, &result)

// ALWAYS
defer resp.Body.Close()
decoder := json.NewDecoder(resp.Body)
decoder.Decode(&result)
```

Applies to every Ollama API call: generate, embed, describe.

---

## 3. Worker jobs — store path only, never content

```go
// NEVER
type Job struct {
    ID      string
    Type    string
    Content string  // never store content in job
}

// ALWAYS
type Job struct {
    ID       string
    Type     string
    NotePath string  // read content only when processing
}

// in worker
func (w *Worker) processJob(job Job) error {
    content, err := os.ReadFile(job.NotePath)
    defer func() { content = nil }()  // eligible for GC after processing
    // process...
}
```

---

## 4. HTTP request body — always limit and close

```go
// every capture handler must have both of these
r.Body = http.MaxBytesReader(w, r.Body, 20<<20)  // 20MB limit
defer r.Body.Close()

// decode streaming, never ReadAll
var req CaptureRequest
json.NewDecoder(r.Body).Decode(&req)
```

---

## 5. String building — always strings.Builder in loops

```go
// NEVER
output := ""
for _, r := range results {
    output += formatResult(r)
}

// ALWAYS
var b strings.Builder
b.Grow(len(results) * 200)
for _, r := range results {
    b.WriteString(formatResult(r))
}
output := b.String()
```

Applies to: search result formatting, reindex progress output, any string built across iterations.

---

## 6. Defer in loops — never defer inside a loop

```go
// NEVER
for _, path := range paths {
    f, _ := os.Open(path)
    defer f.Close()  // all stay open until function returns
    process(f)
}

// ALWAYS — close explicitly or use inner func
for _, path := range paths {
    f, _ := os.Open(path)
    process(f)
    f.Close()
}
```

---

## 7. sync.Pool for embedding buffers during reindex

```go
var embPool = sync.Pool{
    New: func() any {
        s := make([]float32, 768)
        return &s
    },
}

// in reindex loop
buf := embPool.Get().(*[]float32)
copy(*buf, newEmbedding)
storeEmbedding(notePath, *buf)
*buf = (*buf)[:768]
embPool.Put(buf)
```

Only needed in reindex — not in normal capture flow.

## 8. GOMEMLIMIT in launchd plist

In the launchd plist generated by `khayal init`:

```xml
<key>EnvironmentVariables</key>
<dict>
    <key>GOMEMLIMIT</key>
    <string>512MiB</string>
    <key>GOGC</key>
    <string>200</string>
</dict>
```

`GOMEMLIMIT=512MiB` — hard ceiling, GC gets aggressive before this.
`GOGC=200` — collect less frequently for a long-lived server process.

## 9. pprof in debug mode only

```go
import _ "net/http/pprof"

func startDebugServer(cfg Config) {
    if !cfg.Debug { return }
    go func() {
        log.Println(http.ListenAndServe("localhost:6060", nil))
    }()
}
```

Add `debug: false` to config.yaml. Never enable in production.

---

## Rules

1. Never `io.ReadAll` on any HTTP response body — always stream decode
2. Never accumulate all embeddings in a slice — always stream with fixed heap
3. Never store note content in queue — store path only
4. Every `http.Request.Body` gets `MaxBytesReader` + `defer Close()`
5. Every `http.Response.Body` gets `defer Close()`
6. Never `defer` inside a loop
7. Always `strings.Builder` for string concatenation in loops
8. `GOMEMLIMIT=512MiB` in launchd plist always
